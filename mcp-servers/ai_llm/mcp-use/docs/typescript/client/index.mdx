Use the TypeScript MCP client when your app needs to connect to one or more MCP servers. `MCPClient` stores server configuration, and each `MCPSession` is one live connection where you list tools, call tools, read resources, and fetch prompts.

This page helps you choose the right client path and make your first call. For every constructor option and method signature, see the [`MCPClient` API reference](/typescript/api-reference/client/mcp-client) and [`MCPSession` API reference](/typescript/api-reference/client/mcp-session).

## Choose your client path

| You are building | Start here | Import path |
| --- | --- | --- |
| A Node.js script, service, or agent integration | [`MCPClient` API reference](/typescript/api-reference/client/mcp-client) | `mcp-use` |
| A browser app without React | [Environments](/typescript/client/environments) | `mcp-use/browser` |
| A React app that manages MCP connections in UI | [React integration](/typescript/client/usemcp) | `mcp-use/react` |

Node.js clients support HTTP and Standard I/O servers. Browser and React clients support HTTP servers.

## Connect and call a tool

This Node.js example connects to one HTTP server, lists available tools, calls one tool, and closes the connection.

```typescript
import { MCPClient } from "mcp-use";

const client = new MCPClient({
  mcpServers: {
    demo: {
      url: "http://localhost:3000/mcp",
    },
  },
});

try {
  const session = await client.createSession("demo");

  const tools = await session.listTools();
  console.log("Tools:", tools.map((tool) => tool.name));

  const result = await session.callTool("echo", {
    message: "Hello from mcp-use",
  });

  const firstBlock = result.content[0];
  if (firstBlock?.type === "text") {
    console.log(firstBlock.text);
  }
} finally {
  await client.close();
}
```

Replace `echo` and its arguments with a tool returned by `listTools()`. `createSession("demo")` throws when `demo` is not configured.

## How the client is organized

`MCPClient` is the configuration and lifecycle object. Create it once, then use it to add servers, open sessions, close sessions, or enable callbacks such as sampling and elicitation.

`MCPSession` is the protocol object. Use a session to call tools, read resources, fetch prompts, subscribe to notifications, and set roots for one server.

```typescript
const url = "http://localhost:3000/mcp";
const client = new MCPClient({ mcpServers: { demo: { url } } });
const session = await client.createSession("demo");

await session.listTools();
await session.callTool("tool-name", {});

await client.closeSession("demo");
```

To connect to every configured server, use `createAllSessions()`. To retrieve a session you already opened, use `getSession(name)`, which returns `null` when no session exists.

## Configure servers

Each entry under `mcpServers` is named by you. The fields inside the entry determine the transport.

```typescript
import { MCPClient } from "mcp-use";

const client = new MCPClient({
  mcpServers: {
    productionApi: {
      url: "https://api.example.com/mcp",
      authToken: process.env.MCP_TOKEN,
    },
    localFiles: {
      command: "npx",
      args: ["-y", "@modelcontextprotocol/server-filesystem", "./"],
    },
  },
});
```

Use `url` for a remote HTTP server. Use `command` and `args` for a local Standard I/O server. See the [`MCPClient` API reference](/typescript/api-reference/client/mcp-client) for every server config field, constructor option, callback, and lifecycle method.
