Tools are functions exposed by an MCP server. In the TypeScript client, you use an initialized `MCPSession` to list available tools and call a tool by name.

If you want to define tools on a server, see [Server tools](/typescript/server/tools).

## List tools

Call `session.listTools()` after you open a session. It fetches the current tool list from the server.

```typescript
import { MCPClient } from "mcp-use";

const client = new MCPClient({
  mcpServers: {
    demo: {
      url: "http://localhost:3000/mcp",
    },
  },
});

try {
  const session = await client.createSession("demo");
  const tools = await session.listTools();

  for (const tool of tools) {
    console.log(tool.name, tool.description);
    console.log(tool.inputSchema);
  }
} finally {
  await client.close();
}
```

Use `session.tools` only when you want the connector's cached tool list. Use `listTools()` when you want fresh data.

## Call a tool

Call `session.callTool(name, args)` with the tool name and an object that matches the tool's input schema.

```typescript
const result = await session.callTool("read_file", {
  path: "/tmp/report.txt",
});

if (result.isError) {
  console.error("Tool returned an error:", result.content);
} else {
  for (const block of result.content) {
    if (block.type === "text") {
      console.log(block.text);
    }
  }
}
```

`callTool()` returns a `CallToolResult` from the MCP SDK. `content` is an array of typed content blocks, so narrow each block by `block.type` before reading fields such as `text`.

## Set request timeouts

Pass MCP request options as the third argument to `callTool()`.

```typescript
const result = await session.callTool(
  "long_running_task",
  { input: "large dataset" },
  {
    timeout: 300_000,
    resetTimeoutOnProgress: true,
    maxTotalTimeout: 900_000,
  }
);
```

Use these options for long-running tools:

| Option | What it does |
| --- | --- |
| `timeout` | Maximum time, in milliseconds, for a request before it times out. |
| `resetTimeoutOnProgress` | Resets the timeout when the server sends progress notifications. |
| `maxTotalTimeout` | Sets an overall maximum time even when progress resets the timeout. |
| `signal` | Cancels the request with an `AbortSignal`. |

## Handle missing sessions

`getSession(name)` returns `null` if the session has not been opened. Use `requireSession(name)` when a missing session should throw.

```typescript
let session = client.getSession("demo");

if (!session) {
  session = await client.createSession("demo");
}

const result = await session.callTool("echo", {
  message: "hello",
});
```
