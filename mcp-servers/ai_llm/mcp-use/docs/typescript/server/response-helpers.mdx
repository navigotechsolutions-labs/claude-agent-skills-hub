Response helpers create MCP-compatible results for tools, resources, and prompts. Use this page to choose the helper. Use the [Response helpers API reference](/typescript/api-reference/server/response-helpers) for exact signatures, parameters, return shapes, MIME types, and edge cases.

## Pick a helper

| Use case | Helper | Notes |
| --- | --- | --- |
| Return normal model-readable text | `text()` | Best default for simple tool and prompt output. |
| Return Markdown | `markdown()` | Use when the client should render formatted text. |
| Return structured JSON | `object()` | Adds structured content and a JSON text representation. |
| Return an array | `array()` | Wraps the array in a `data` field for structured content. |
| Return a failed tool result | `error()` | Sets the MCP tool-result error flag. |
| Return several content blocks | `mix()` | Combines text, media, resources, and structured content. |
| Embed a resource reference | `resource()` | Use when a tool result should point to readable resource content. |
| Return an MCP App widget | `widget()` | Sends widget props as structured content and model-facing text through `output`. |
| Return media or binary content | `image()`, `audio()`, `binary()` | Use the exact MIME-type behavior from the API reference. |

## Common patterns

Use `text()` for the smallest possible tool result:

```typescript
import { text } from "mcp-use/server";

server.tool(
  {
    name: "get-time",
    description: "Return the current server time",
  },
  async () => text(new Date().toISOString()),
);
```

Use `object()` when the client or widget needs structured data:

```typescript
import { object } from "mcp-use/server";

server.tool(
  {
    name: "get-status",
    description: "Return server status",
  },
  async () =>
    object({
      status: "ok",
      checkedAt: new Date().toISOString(),
    }),
);
```

Use `widget()` when a tool returns an MCP App widget:

```typescript
import { text, widget } from "mcp-use/server";

return widget({
  props: { results },
  output: text(`Found ${results.length} results`),
});
```

The model sees the `output` content. The widget reads `props` through `useWidget().props`.
