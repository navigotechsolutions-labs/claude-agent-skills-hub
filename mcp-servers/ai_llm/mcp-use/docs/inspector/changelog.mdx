  ## MCP Apps runtime, Auto connections & host context styles
  Major release paired with `mcp-use` v1.33.0 — widgets now render with MCP Apps as the primary runtime, new connections default to Auto mode with proxy fallback, and host context style variables are applied while the Apps SDK preview tab remains for `window.openai`-only widgets.

- **New**: widgets render with the MCP Apps bridge as the primary runtime and apply host context style variables; the Apps SDK ("Component (Apps SDK)") preview tab remains for widgets that only speak `window.openai` (`#1739`)
- **Enhancement**: new connections default to Auto mode, trying a direct browser connection first and falling back to the configured Inspector proxy when CORS or other proxy-resolvable connection errors occur; Direct and Proxy remain available as advanced overrides (`#1795`)
- **Fix**: OAuth proxy fetch is scoped per server configuration, so connecting one server Via Proxy no longer mutates global `fetch`, changes unrelated requests, or affects other servers configured for Direct mode (`#1770`)
- **Fix**: stale saved auto-connect entries are replaced when the advertised transport changes, so embedded Inspector instances stop retrying deprecated SSE connections after switching to streamable HTTP (`#1792`)
- **Fix**: signed-in users on the hosted free tier are no longer shown the "sign in to increase your limits" prompt — the free-tier upgrade chrome only renders for anonymous visitors (`#1767`)
- **Fix**: the inspector CLI handles `--version` and unknown options and reads its version from `package.json`, so canary builds and CLI version output stay in sync (`#1722`)
- **Update**: Bumped `vite` to 8.0.16 (`#1742`)
- **Update**: Peer dependency on `mcp-use` moves to `>=1.33.0-canary.0`
- **Update**: Updated mcp-use to v1.33.0
</Update>

  ## Local LLM provider, image tool results & widget display modes
  Major release paired with `mcp-use` v1.29.0: chat against a local Ollama daemon, MCP image tool results forwarded to vision models as real images, an MCP-server reconnect banner, and a reworked widget fullscreen / picture-in-picture path  -  plus a large dead-code cleanup.

- **New**: Ollama as a local LLM chat provider  -  local model discovery and streaming chat, talking to the daemon directly over CORS (no server-side proxy) with an `OLLAMA_ORIGINS` hint surfaced when the browser fetch is blocked (`#1395`)
- **New**: MCP-server reconnect banner  -  chat recognises the cloud's `mcp_auth_required` 401 (e.g. an expired upstream OAuth token) and renders an inline reconnect banner for the affected server that refreshes the token in place, instead of pointing users at the Manufact account login (`#1485`)
- **New**: copy button on user messages, matching the assistant-message UX (`#1556`)
- **Enhancement**: `mcp-use client screenshot` auto-sizes to the widget's natural dimensions when `--width`/`--height` are omitted and handles external MCP server URIs (`ui://<server>/<name>.html`) (`#1579`)
- **Enhancement**: snappier tab-switch animation timing (`#1544`)
- **Fix**: MCP image tool results are forwarded to the model as real image content instead of base64-encoded JSON  -  image bytes route through each provider's vision channel (Anthropic `tool_result` image blocks; OpenAI / Google follow-up user turns), and text-only results now reach the model unwrapped (`#1487`)
- **Fix**: widget fullscreen renders above embedded host chrome  -  native `requestFullscreen()` on the widget shell with an exit navbar, picture-in-picture portaled to `document.body` at `z-[100]`, and AppBridge reconnect after the sandbox iframe remounts (`#1605`)
- **Fix**: stop `ToolInputForm` from writing an invalid per-property `required: boolean` onto the live tool `inputSchema` during render, which made Anthropic reject the chat request with `tools.0.custom.input_schema invalid` after a tool was opened in the Tools tab (`#1545`)
- **Fix**: keep per-tab count badges visible and right-aligned in collapsed header tabs (`#1490`)
- **Fix**: propagate server alias updates through the MCP server wrapper so the tile heading reflects a renamed alias immediately (`#1539`)
- **Fix**: replace the trailing-slash regex in `normalizeOllamaBaseUrl` with a linear scan, resolving a CodeQL polynomial-regex alert (`#1534`)
- **Chore**: add an OAuth redirect e2e against an emulated Google issuer, exercising both Direct and Via Proxy connection types (`#1481`, `#1609`)
- **Chore**: remove 18 unused source files, stop generating the unused client-side `version.ts`, and drop dead barrel re-exports flagged by Knip (`#1510`, `#1533`, `#1607`)
- **Update**: Updated mcp-use to v1.29.0
</Update>
